# Recording checklist

## Before
- [ ] `npm install` completed; `npm run build` passes; `npm test` green (60/60).
- [ ] `npm run dev`, browser at `http://localhost:5173`, window sized **1600×900**
      (16:9). Hide bookmarks bar; full-screen the content.
- [ ] Zoom 100%. Dark room / dark desktop background looks best with the theme.
- [ ] Decide: reduced-motion OFF for the recording (default) for smooth token
      travel. (`prefers-reduced-motion` collapses animation if a viewer needs it.)
- [ ] Audio: script from `docs/DEMO_3_MINUTES.md`.

## Capture
- [ ] Start on Scenario A, paused. Read the hook (15s).
- [ ] Click **Run 3-minute demo**. Let it auto-play; narrate each beat.
- [ ] If a beat needs dwell, press **Space** to pause, then Space to resume, or
      use **←/→** to step between beats.
- [ ] Ensure the **boundary gate opening** (A) and the **BLOCKED stamp** (C) are
      clearly on screen.
- [ ] Let the **summary overlay** appear at the end; hold on it for the close.

## Stills (already generated in `docs/screenshots/`)
- [ ] 01 clean intake · 02 clean crossed · 03 changed IBAN · 04 document
      injection · 05 integrity failed · 06 replay blocked.
- [ ] Regenerate via deep-links if needed (see DEMO_3_MINUTES.md).

## Truthfulness (do not claim)
- [ ] Do **not** say "safe", "fraud-free", "paid", or "Qonto-approved".
- [ ] Do **not** present CFO/CEO/Designated Approver as native Qonto roles.
- [ ] State clearly: **synthetic data**, **no Qonto write**, **no money moved**,
      **SCA remains in Qonto**.
